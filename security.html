Security

How WorklineIQ protects your company's field data, from the database to the device.

Our commitment Your data belongs to you. We process it on your behalf using industry-standard security controls. We don't sell it, share it with advertisers, or use it for any purpose outside of delivering the platform to your team.

Security Controls

🔒
Row-Level Security (RLS)
Enabled on every database table. Your company's data is isolated at the database level — no query can return another tenant's records. Data isolation is enforced by the database, not just the application layer.
Database-enforced
🔐
Encryption in Transit
All communication between your devices and our servers is encrypted via TLS 1.2 or higher. HTTPS is enforced across all endpoints. No data travels unencrypted.
TLS 1.2+
💾
Encryption at Rest
All stored data is encrypted using AES-256, managed at the infrastructure level by Supabase. This includes the database, backups, and file storage.
AES-256
👤
Role-Based Access Control
25+ granular permission flags control what each user role can see and do. Permissions are enforced server-side via database triggers — not just in the UI.
Server-side enforced
🔑
Authentication & Sessions
Admin and office users authenticate via Supabase Auth with JWT tokens (1-hour expiry). Field crew access uses server-side PIN validation — no personal accounts or email addresses required.
JWT + PIN
📱
Offline Data Handling
Field entries queued offline are stored locally in IndexedDB on the device. The queue syncs automatically when connectivity is restored and is cleared upon logout.
Cleared on logout
